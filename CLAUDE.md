# CLAUDE.md — UL登山ギアリスト計算ツール（日本語版LighterPack）

> このファイルはClaude Codeへの引き継ぎ文脈。プロジェクト直下に置く。

## 1. 一言でこれは何か
日本語ネイティブの **UL/軽量登山者向けギアリスト＋重量計算 Webツール**。
手持ちギアを「選ぶだけ」で登録し、**ベースウェイト／パックウェイト／スキンアウト**を自動算出。
円グラフでカテゴリ別内訳を表示し、共有もできる。
+ ギア個別ページ（本物のスペックデータ）でSEO集客し、購入リンク（アフィリエイト）で収益化。

LighterPack（英語の定番ツール）の「日本版＋選ぶだけ」ポジション。

## 2. なぜ作るか（検証済みの根拠）
- **ロケールの穴**：英語LighterPackは定番だが日本語ネイティブ版が存在しない。JPのUL勢はEN版かExcel/Numbersで我慢している。「WEBアプリ化してほしい」という公開された要望が複数確認できた。
- **堀（参入障壁）**：日本語ギアDB＝重量データの収集・維持が面倒。この面倒さがそのまま堀になる。
- **市場性**：UL勢は人数は小さいが高額ギアを買う層（アフィリ単価が出る）。1g単位で実測する文化がある。
- **具体的な痛み**：「ベースウェイトに着替えやトレッキングポールを含むか」が曖昧で繰り返し議論になっている → フラグ＋自分ルール切替で解決できる。

## 3. ターゲット（ビーチヘッド）
**UL/軽量登山勢**（グラム単位で計る層）。オートキャンプ勢は対象外。

## 4. MVP（最小スライス）
### コア＝勝負どころ「選ぶだけ」
- ギアリストを作る画面。入力は2通り：
  1. **日本語ギアDBから選ぶ**（重量が自動で入る）← ここが体験の生命線
  2. **手入力**（DBに無い／中華・ノーブランド／自作）
- 各アイテムにフラグ：`消耗品`（水・食料・燃料）／`着用品`（常時着用）／数量
- 自動算出：
  - ベースウェイト＝消耗品・着用品を除く
  - パックウェイト＝消耗品込み
  - スキンアウト＝全部
  - ※「何を含む」は自分ルールでトグル切替できるようにする
- カテゴリ別の**円グラフ内訳**
- 目標達成メーター（例：4.5kg）
- **共有**：リストをURLエンコード or localStorageで保持し、共有リンクを生成

## 5. スコープ
### やる（MVP）
計算機 ＋ 円グラフ ＋ 共有 ＋ シードDB数十点 ＋ ギア個別ページ少数

### やらない（後回し）
アカウント／ログイン、YAMAP・ヤマレコ連携、DB全網羅、ネイティブアプリ化（まずWeb）、課金（プレミアム）

## 6. 技術スタック
- **Next.js（静的書き出し / SSG）＋ Vercel 無料枠**
- ギアDB ＝ **静的JSON**（リポジトリ内で管理、read-only）
- ユーザーのリスト ＝ **localStorage ＋ URLエンコード共有**
- 認証・バックエンドDB・課金は**無し**（サーバーコスト0・壊れにくい・放置運用向き）

## 7. ギアDBスキーマ（案）
```json
{
  "id": "yamatomichi-mini2",
  "brand": "山と道",
  "name": "MINI2",
  "category": "backpack",
  "weight_g": 448,
  "capacity": "30-40L",
  "price_jpy": 39600,
  "source_type": "manufacturer",   // manufacturer | retailer | measured
  "confidence": "high",            // high | medium | low
  "source_url": "https://...",
  "affiliate_url": "https://...",
  "lighter_alternatives": ["..."]  // 軽量代替候補のid配列
}
```
カテゴリ例：backpack / shelter / sleep / cook / clothing / electronics / hydration / misc

## 8. 設計ガードレール（必ず守る）
- **重量データは捏造禁止。** 必ず一次情報（メーカー／店舗ページ）由来。AI抽出は下書きで、人が確認してから登録。
- 各データに `source_type`／`confidence`／`source_url` を必ず持たせる（数字のズレは信頼を一発で失う）。
- 中華・ノーブランドは**実測レンジ**＋出典。ユーザーの実測入力も受け付ける。
- アフィリエイトリンクは各プラットフォームの規約を守る。
- **シードDBは「UL勢が実際に使う定番」に当てる**（適当に埋めない）。"選ぶだけ"が楽でないと負ける。

## 9. 集客＆収益（時間軸は正直に）
- ギア個別ページ／カテゴリ別比較ページでSEO → 計算機へ送客 → アフィリ＋AdSense
- 各ページが固有の実データを持つので、Googleの量産コンテンツ規制（scaled content abuse）に強い
- SEOは効くまで数ヶ月。「出発前に種まき → 滞在中に育つ」前提。早期収益は小さいと割り切る

## 10. ビルド順（出発前にここまで）
1. リポジトリ作成 ＋ Next.js雛形 ＋ Vercel連携
2. ギアDBスキーマ確定 ＋ シード数十点（出典付き、定番ギアから）
3. 計算機UI（選ぶ／手入力 → ベース/パック/スキンアウト自動 → 円グラフ → 達成メーター）
4. 共有（URLエンコード / localStorage）
5. ギア個別ページをDBから生成（SEO ＋ アフィリ）
6. Vercelデプロイ → Search Consoleにsitemap提出

## 11. 検証（作りながら）
シード数十点でα版 → 自分＋UL勢数人に使ってもらい「選ぶだけは本当に楽か」「LighterPack/Excelから乗り換えるか」を聞く。**3人が「乗り換える」と言えば本物。**

## 12. 正直なリスク（前提として持っておく）
- 市場が小さい（UL勢は数千〜数万規模かも）＝大きな収益は期待しすぎない
- 無料の名作LighterPackからの乗り換え障壁 → "選ぶだけ"の体験で殴る
- YAMAP/ヤマレコが同機能を実装したら脅威（今は無い＝窓が開いている）
- SEOは遅く不確実
- → 位置づけは「学ぶ実験 ＋ ポートフォリオ ＋ 小さなストック収入」

## 13. 用語メモ
- **ベースウェイト**：水・食料・燃料などの消耗品を除いた装備重量（UL基準 4.5kg以下）
- **パックウェイト**：消耗品を含めた、背負う総重量
- **スキンアウト**：着用品・身につけるもの含む全部
